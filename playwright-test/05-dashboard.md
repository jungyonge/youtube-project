# 05. 대시보드 테스트 (단위)

## 목적
대시보드 페이지의 레이아웃, 영상 생성 폼, 작업 목록 표시를 테스트합니다.

## 사전 조건
- 로그인 완료 상태

---

## 테스트 케이스 1: 대시보드 페이지 레이아웃 확인

### 프롬프트

```
Playwright MCP를 사용하여 다음 테스트를 수행해줘:

1. test@example.com / testpassword123 으로 로그인
2. browser_snapshot으로 대시보드 상태 캡처
3. 다음 요소가 존재하는지 확인:
   - "대시보드" 제목 (h1, text-2xl font-bold)
   - 데스크탑(lg 이상): 2컬럼 레이아웃
     - 좌측: "영상 생성" 카드 (CardTitle)
     - 우측: "내 영상" 섹션 제목 + JobList
   - "주제" 텍스트 에어리어 (placeholder: "영상으로 만들 주제를 입력하세요 (5~200자)")
   - "소스" 영역 + 소스 입력 필드
   - 스타일 선택 버튼 4개: "정보 전달", "스토리텔링", "튜토리얼", "오피니언"
   - "고급 설정" 토글 버튼 (ChevronDown 아이콘)
   - "영상 생성 시작" 제출 버튼
4. 결과를 정리해서 알려줘

(자동 수정 규칙: 00-setup.md Auto-Fix Protocol 적용)
```

### 기대 결과
- 2컬럼 레이아웃 (좌: 생성 폼, 우: 영상 목록)

---

## 테스트 케이스 2: 고급 설정 토글 확인

### 프롬프트

```
Playwright MCP를 사용하여 다음 테스트를 수행해줘:

(로그인된 상태에서 대시보드)

1. browser_snapshot으로 현재 상태 확인 — 고급 설정이 접혀 있는지
2. "고급 설정" 버튼(ChevronDown)을 browser_click으로 클릭
3. browser_snapshot으로 펼쳐진 상태 확인
4. 다음 고급 설정 항목이 표시되는지 확인:
   - "영상 길이: 10분" 슬라이더 (min: 10, max: 15, step: 1)
   - "음성" 드롭다운: Alloy(중성), Echo(남성), Fable(남성), Onyx(남성,저음), Nova(여성), Shimmer(여성)
   - "언어" 드롭다운: 한국어, English, 日本語
   - "자막 포함" 스위치 (기본: ON)
   - "배경 음악" 스위치 (기본: ON)
   - "비용 예산 ($)" 입력 필드 (기본: 2, min: 0.5, max: 10, step: 0.5)
   - "자동 승인" 스위치 (기본: ON)
     - OFF 시 "대본 생성 후 직접 검토하고 승인해야 영상 생성이 진행됩니다." 설명 텍스트
   - "추가 지시사항" 텍스트 에어리어
5. 다시 "고급 설정" 버튼(ChevronUp)을 클릭하여 접히는지 확인
6. 결과를 정리해서 알려줘

(자동 수정 규칙: 00-setup.md Auto-Fix Protocol 적용)
```

### 기대 결과
- 토글 ON/OFF 동작
- 자동 승인 OFF 시 안내 문구 표시

## 테스트 케이스 3: 스타일 선택 동작 확인

### 프롬프트

```
Playwright MCP를 사용하여 다음 테스트를 수행해줘:

3. "스토리텔링" 버튼을 클릭
4. browser_snapshot으로 "스토리텔링"이 선택 상태로 변경, "정보 전달"은 해제인지 확인

7. 각 스타일 카드에 아이콘과 설명이 올바르게 표시되는지:
   - 정보 전달: BookOpen + "객관적 사실 중심"
   - 스토리텔링: Clapperboard + "이야기 구조"
   - 튜토리얼: GraduationCap + "단계별 설명"
   - 오피니언: MessageSquare + "관점 제시"
8. 결과를 정리해서 알려줘

(자동 수정 규칙: 00-setup.md Auto-Fix Protocol 적용)

## 테스트 케이스 10: 소스 URL 유효성 검증

### 프롬프트

```
Playwright MCP를 사용하여 다음 테스트를 수행해줘:

(로그인된 상태에서 대시보드)

1. 주제: "URL 유효성 테스트 영상입니다" 입력
2. 소스 타입: "블로그" (기본)

   - URL 형식 에러 메시지가 표시되는지

6. 소스 URL을 "https://example.com/valid" 로 수정
7. 정상 제출 가능한지 확인
8. 결과를 정리해서 알려줘

(자동 수정 규칙: 00-setup.md Auto-Fix Protocol 적용)
```

### 기대 결과

## 테스트 케이스 11: 비용 예산 경계값 테스트

### 프롬프트

```
Playwright MCP를 사용하여 다음 테스트를 수행해줘:

6. 비용 예산에 "11" 입력 → 에러 확인 (최대 초과)
