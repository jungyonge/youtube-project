# 19. 반응형 및 크로스 브라우저 테스트

## 목적
다양한 화면 크기(모바일, 태블릿, 데스크탑)에서의 레이아웃과 기능 동작을 테스트합니다.

---

## 테스트 케이스 1: 모바일(375x812) — 로그인/회원가입

### 프롬프트

```
Playwright MCP를 사용하여 다음 반응형 테스트를 수행해줘:

1. browser_resize로 모바일 크기(375x812 — iPhone X) 설정
2. browser_navigate로 http://localhost:5173/login 접속
3. browser_snapshot으로 확인:
   - 로그인 카드가 화면 너비에 맞게 조정되는지
   - 입력 필드가 터치 가능한 크기(최소 44px 높이)인지
   - 버튼이 화면 너비에 맞게 확장되는지 (w-full)
   - 텍스트가 잘리지 않는지
4. /register 페이지도 동일하게 확인
5. 로그인 후 모바일 레이아웃 확인
6. browser_resize로 데스크탑(1280x720)으로 복원
7. 결과를 정리해줘

(자동 수정 규칙: 00-setup.md Auto-Fix Protocol 적용)
```

### 기대 결과
- 모바일에서 터치 친화적 UI

## 테스트 케이스 4: 모바일 — 작업 상세 페이지

### 프롬프트

```
Playwright MCP를 사용하여 다음 반응형 테스트를 수행해줘:

1. browser_resize로 모바일 크기(375x812) 설정
2. 작업 상세 페이지로 이동
3. browser_snapshot으로 확인:
   - 파이프라인 9단계 인디케이터가 모바일에서 어떻게 표시되는지
     (수평 스크롤, 축소, 또는 세로 배치)
   - 상태 Badge, 비용 배지가 잘리지 않는지
   - 탭(진행 상세/대본/결과)이 정상 동작하는지
   - 액션 버튼(취소, 재시도 등)이 터치 가능한 크기인지
4. "진행 상세" 탭의 테이블이 모바일에서 어떻게 표시되는지:
   - 수평 스크롤 여부
   - 핵심 컬럼만 표시 여부
5. browser_resize로 데스크탑 복원
6. 결과를 정리해줘

(자동 수정 규칙: 00-setup.md Auto-Fix Protocol 적용)
```

### 기대 결과

4. 하단 버튼 3개("거부", "수정 요청", "승인")가:
   - 터치 가능한 크기인지

   - 화면 너비 내에 맞는지
5. browser_resize로 데스크탑 복원
6. 결과를 정리해줘

(자동 수정 규칙: 00-setup.md Auto-Fix Protocol 적용)
