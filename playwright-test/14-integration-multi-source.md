# 14. 통합 테스트: 다중 소스 및 고급 설정 영상 생성

## 목적
여러 소스 타입 조합, 소스 편집(추가/삭제/타입변경), 고급 설정 활용 시나리오를 E2E로 테스트합니다.

---

## 통합 시나리오 1: 3개 소스 타입 조합 생성

### 프롬프트

```
Playwright MCP를 사용하여 다음 통합 테스트를 수행해줘:

### Step 1: 로그인 후 대시보드

### Step 2: 다중 소스 영상 생성
1. 주제: "2024년 한국 스타트업 생태계 현황과 전망"
2. 첫 번째 소스:
   - 타입: "블로그" (기본값)
   - URL: "https://example.com/startup-ecosystem"
3. "소스 추가" 클릭 → 두 번째 소스:
   - 타입 드롭다운 "뉴스" 선택
   - URL: "https://example.com/startup-news"
4. "소스 추가" 클릭 → 세 번째 소스:
   - 타입 드롭다운 "유튜브" 선택
   - URL: "https://youtube.com/watch?v=startup-analysis"
5. browser_snapshot으로 3개 소스 모두 입력 확인:
   - 각 소스의 타입 드롭다운 값
   - 각 소스의 URL 입력 값
   - 모든 삭제 버튼 활성화 (3개 > 1)
   - "소스 추가" 버튼 여전히 표시 (3 < 10)

### Step 3: 고급 설정
6. "고급 설정" 펼치기
7. 스타일: "스토리텔링"
8. 음성: "Echo (남성)"
9. 영상 길이 슬라이더 15분으로 조정
10. 비용 예산: $5.00
11. 추가 지시사항: "각 소스의 핵심 인사이트를 비교 분석하는 형태로 구성해주세요"
12. browser_snapshot으로 전체 설정 확인

### Step 4: 생성 및 확인
13. "영상 생성 시작" 클릭
14. 토스트 + 상세 페이지 이동 확인
15. 작업이 생성되었는지 확인
16. 10초 후 browser_snapshot으로 파이프라인 진행 확인

각 단계의 결과를 정리해줘.

(자동 수정 규칙: 00-setup.md Auto-Fix Protocol 적용)
```

### 기대 결과
| Step | 기대 결과 |
|------|-----------|
| 2 | 3개 소스 정상 추가 |
| 3 | 고급 설정 반영 |
| 4 | 작업 생성 + 파이프라인 시작 |

## 통합 시나리오 2: 소스 추가/삭제/편집 후 생성

### 프롬프트

```
Playwright MCP를 사용하여 다음 통합 테스트를 수행해줘:

### Step 1: 소스 편집 시나리오
1. 주제: "프로그래밍 언어 트렌드 2024"
2. 기본 소스(블로그) URL: "https://example.com/programming-trends"
3. 소스 2개 추가 → 총 3개
4. browser_snapshot으로 3개 확인

### Step 2: 삭제 → 추가
5. 두 번째 소스 삭제 (Trash2 클릭)
6. browser_snapshot → 2개 확인
7. "소스 추가" 클릭
8. 새 소스 타입: "직접 입력" (custom_text) 선택
9. Textarea가 나타나는지 확인
10. 텍스트 입력: "Python, JavaScript, Rust가 2024년 가장 주목받는 언어입니다."
11. browser_snapshot으로 최종 소스 구성 확인:
    - 소스1: blog + URL
    - 소스2: custom_text + Textarea

### Step 3: 생성
12. "영상 생성 시작" 클릭
13. 작업 생성 성공 확인

각 단계의 결과를 정리해줘.

(자동 수정 규칙: 00-setup.md Auto-Fix Protocol 적용)
```

### 기대 결과
| Step | 기대 결과 |
|------|-----------|
| 1-2 | 소스 추가/삭제 정상 |
| 3 | 혼합 소스(URL + custom_text)로 생성 성공 |

## 통합 시나리오 3: 소스 10개 제한 확인 후 생성

### 프롬프트

```
Playwright MCP를 사용하여 다음 통합 테스트를 수행해줘:

### Step 1: 소스 10개까지 추가
1. 주제: "소스 제한 테스트"
2. "소스 추가" 버튼을 9회 클릭 (기본 1 + 9 = 10개)
3. 각 추가 시 소스 개수 확인
4. 10개에 도달 시 "소스 추가" 버튼이 **사라지는지** 확인

### Step 2: 하나 삭제 후 버튼 복귀
5. 소스 하나 삭제 → 9개
6. "소스 추가" 버튼이 다시 나타나는지 확인

### Step 3: 최종 생성 (선택)
7. 적절한 수(3개 정도)로 줄인 후 생성 시도

각 단계의 결과를 정리해줘.

(자동 수정 규칙: 00-setup.md Auto-Fix Protocol 적용)
```

### 기대 결과
| Step | 기대 결과 |
|------|-----------|
| 1 | 10개 도달 시 추가 버튼 숨김 |
| 2 | 삭제 후 버튼 복귀 |
